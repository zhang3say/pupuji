# 噗噗记 — 产品需求文档 (PRD)

## 一、产品概述

### 1.1 产品定位

「噗噗记」是一款面向职场人的带薪拉屎追踪 APP，以「带薪时间/金钱追踪」为核心，「如厕记录 + 趣味化」为辅。通过将如厕时间换算为「公司为你付的钱」，让职场人在蹲坑中获得微小的快乐与满足感。

### 1.2 核心理念

**「每次噗噗，都是财富」**

将「臭」的事物可爱化，用卡皮巴拉（水豚）泡温泉的意象替代直接的马桶场景，营造松弛、治愈、佛系的情绪氛围。

### 1.3 目标用户

- 核心：有幽默感的职场打工人（20-35 岁）
- 场景：办公室如厕时使用

## 二、品牌设计

### 2.1 名称

**噗噗记** — 「噗噗」可爱化如厕行为，「记」兼具记录与记账之意。

### 2.2 IP 角色：卡皮巴拉

- 选择理由：佛系、情绪稳定、泡温泉的视觉符号天然契合「蹲坑放松」
- 视觉锚点：圆滚滚的身体、眯眼放松表情、头顶小毛巾、泡在温泉（马桶）里
- 风格：温泉自然风，治愈松弛，与职场紧张感形成反差

### 2.3 配色方案

| 角色 | 色值 |
|------|------|
| 主色（暖棕） | `#B8956A` |
| 背景（米色） | `#FFF8F0` |
| 辅助（淡绿） | `#C5D8B5` |
| 强调（温泉蓝） | `#A8D8EA` |
| 文字 | `#4A3728` |

### 2.4 设计资产

- MVP 阶段：基础温泉风配色 + emoji + 极简 UI
- 后续：AI 生成卡皮巴拉素材统一风格

## 三、功能清单

### 3.1 P0（MVP 必需）

#### 3.1.1 用户系统
- 手机号 + 短信验证码登录（免密）
- JWT Token 认证（Bearer）
- 个人中心：头像、昵称、薪资设置、退出登录

#### 3.1.2 薪资设置
- 输入月薪，按 21.75 天/月、8 小时/天换算分钟薪
- 工作天数和工作时长可自定义
- 计算公式：`分钟薪 = 月薪 ÷ 天数 ÷ 小时数 ÷ 60`

#### 3.1.3 计时器
- 「开始噗噗」按钮，一键开始计时
- 暂停/继续：支持中途被叫走
- 结束：停止计时并自动计算本次收益
- 实时显示：计时中显示当前累计金额（`+¥X.XX`）

#### 3.1.4 手动补录
- 事后填开始时长的「手动记录」功能
- 补录条目标记为「手动记录」以示区分

#### 3.1.5 今日汇总
- 今日如厕次数
- 今日总时长
- 今日赚了多少钱
- 「公司今天为你付了 ¥XX.XX」反向视角

#### 3.1.6 历史记录
- 单次记录列表（开始时间、时长、金额、是否为手动补录）
- 按日期分组展示

#### 3.1.7 首次引导
- 三步精美引导页（卡皮巴拉插图）
- 手机号注册
- 填写薪资

### 3.2 P1（后续版本）

- 统计图表：周/月/年的时长与金额柱状图/折线图
- 勋章系统：趣味成就（「今日已释放」「一蹲千金」「厕所哲学家」「碎片化拉屎」「带薪拉屎满月」等）
- 分享卡片：生成精美图片分享朋友圈/微信群
- 匿名排行榜：全国/同城/同行业对比排名
- 知识卡片：蹲坑时看的冷知识
- 提醒功能：蹲太久腿麻提醒、定时提醒

### 3.3 P2（远期迭代）

- 卡皮巴拉皮肤/装扮
- 白噪音（水流声等）
- Lottie 动画（金币掉落、温泉动画、完成庆祝）
- 卡皮巴拉互动对话
- 鸿蒙端

## 四、技术架构

### 4.1 项目结构

```
pupuji/
├── apps/
│   ├── mobile-rn/      # React Native（iOS/Android）
│   └── harmony/        # 鸿蒙原生（ArkTS/ArkUI）
├── server/             # FastAPI 后端
├── packages/
│   └── shared/         # 共享 TypeScript 类型定义、常量
├── pnpm-workspace.yaml
└── package.json
```

- Monorepo 管理：pnpm workspace
- 版本控制：GitHub Flow（main + feature 分支）

### 4.2 技术栈

| 层 | 技术 | 说明 |
|----|------|------|
| 移动端 | React Native | iOS 为主 |
| 状态管理 | Zustand | 轻量简洁 |
| 导航 | React Navigation | RN 导航标准 |
| HTTP | Axios | 网络请求 |
| 动画 | Reanimated 3 + Lottie | UI 动效 |
| 后端 | FastAPI | Python 异步框架 |
| 数据库 | PostgreSQL | 主数据存储 |
| ORM | SQLAlchemy 2.0 | 数据层 |
| 迁移 | Alembic | 数据库版本管理 |
| 缓存 | Redis | 排行榜/验证码 |
| 部署 | ECS + Docker | 手动部署 |
| 测试 | pytest + jest | 关键路径覆盖 |

### 4.3 API 规范

- 风格：务实 REST-ish
- 认证：JWT Bearer Token
- 版本控制：`X-API-Version: 1` Header
- 响应格式：

```json
{
  "code": 0,
  "data": {},
  "message": "ok"
}
```

- API 文档：FastAPI 自动生成 Swagger

### 4.4 开发工具链

| 类别 | 工具 |
|------|------|
| 代码规范（RN） | ESLint + Prettier |
| 代码规范（Python） | Ruff |
| Git Hook | pre-commit |

| 字段 | 类型 | 说明 |

## 七、UX 流程

### 7.1 首次使用

```
打开 APP → 三步引导页 → 手机号注册 → 填写月薪 → 进入首页 → 开始噗噗
```

### 7.2 核心流程

```
点击「开始噗噗」→ 计时开始（卡皮巴拉表情变化）→ 暂停/继续/结束
→ 结束后弹窗展示本次收益 → 首页汇总更新
```

### 7.3 手动补录

```
记录页 → 点击「+」→ 填写开始时间、时长 → 保存 → 标记为手动记录
```

## 九、风险与注意事项

- **应用商店审核**：避免直接使用排泄物相关视觉元素，「噗噗」「温泉」概念降低审核风险
- **隐私合规**：手机号登录需遵守《个人信息保护法》，隐私政策 + 用户协议必备
- **计时精度**：小程序如后续上线，需注意后台限制；RN 端 iOS/Android 无此问题
- **数据安全**：薪资数据敏感，生产环境需数据库加密 + HTTPS
